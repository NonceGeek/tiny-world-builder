## Overview

TinyWorld is a browser-based voxel world builder with an embedded multiplayer MMO layer (worlds rooms), flight simulation, floating islands, and a roadmap toward a full sky-sea survival game ("Skybound"). The served app is a static build from `dist/` (via `./publish.sh`), Netlify-hosted. CodeSurf edits auto-commit; **main auto-pushes to GitHub → Netlify prod** — branches do not guarantee isolation.

## Durable Architecture Facts

**Source shape**
- Entry: `tiny-world-builder.html` — boot config, DOM scaffold, ordered `<script src>` tags
- Styles: `styles/tiny-world.css` (~5.2k lines)
- Modules: `engine/world/00-*.js` through `69-*.js` + `99-late-boot.js` (~40k LoC total)
- All engine modules are **plain classic scripts sharing one global scope** — load order matters; every top-level `const`/`let`/`function` must be unique across all files. A duplicate identifier throws `SyntaxError` and silently kills that module while others keep loading. Prefix module-local globals (e.g. `_sr`, `_fl`, `_la`). New modules expose via `window.__*` from an IIFE.
- Three.js **r128** pinned; vendor self-hosted under `vendor/three/`. Do not bump the version.
- No bundler, no npm runtime dependencies. Static checks: `npm test`. Dist: `npm run build` or `./publish.sh`.

**World data model**
- `world[x][z]` — intent: `{ terrain, terrainFloors, kind, floors }`
- `cellMeshes['x,z']` — render: `{ tile: Group, object: Group|null }`
- **Always mutate via `setCell(x, z, opts)`** — never write `world[x][z]` directly outside init
- `userData.landing` guards drop-in animations — never remove these checks

**Materials**
- `M.*` materials are shared across many meshes — clone before mutating `.color`
- `disposeGroup(group)` disposes geometries but NOT materials (shared); cloned per-instance materials (smoke) self-dispose on death

**Cameras**
- `orthoCam`, `softCam`, `persCam` coexist; `camera` is a swappable reference
- `togglePerspective()` / `setCameraMode()` write to all projections/positions

**LandscapeEngine**
- `LandscapeEngine.js` is a superseded monolith — `getHeight`/chunk-building live in `engine/landscape/*.js` mixins that override it. Edit the mixins, not the monolith. Constructor stays live (hence `VOXEL:true` but old method bodies).

**Build pipeline**
- Dev: `npm run dev` (:3000) serves source directly — no build step needed during iteration
- Prod: `./publish.sh` → `dist/` → Netlify auto-deploys; `vercel.json` + `netlify.toml` both point to `dist/`
- `tools/check.js` fails build on duplicate top-level identifiers across `engine/world/*.js`

## Multiplayer Architecture

- Transport: PartyKit; server at `party/index.js`; deploys via `partykit deploy` (separate from `publish.sh`)
- **New world-room message handlers must go into `onWorldMessage`**, NOT the main `onMessage` (world rooms early-return past it). This trap shipped chat-emotes' emote relay dead in prod.
- `RATE_LIMITS` bucket — test the world message path explicitly when adding new relay messages
- `world.join` presence carries `avatar` descriptor field (needed for per-player voxel skin identity — currently unconnected for peers)
- Local peers need `openMode` (no `WORLDS_JOIN_SECRET`/`SERVICE_TOKEN`) or a signed play token; otherwise bots/clients are observers and render as peers but cannot harvest

## Lobby Systems

### Lobby Population Bots (`tools/lobby-bots.mjs`)

- `npm run bots:lobby`; spawns ambient NPC peers into `tidewater-bay` lobby world
- External Node CLI process (Node 22+, no deps); never in browser build; companion doc: `tools/lobby-bots.README.md`
- Empty-token `observe` role peers **CAN move, chat, emote, and render as peers** — no token minting needed (verified empirically against `party/index.js`)
  - `handleMove` allows `observe`; chat/emote gated on `admitted.has(id)` (set for every role); only harvest is play+profile gated
- LLM: OpenRouter (`OPENROUTER_API_KEY`); default `meta-llama/llama-3.3-70b-instruct:free`; falls back through `:free` model rotation on 429/404/empty; 401/403 disables globally; `:free` enforced on every attempt
- `TW_ORIGIN` required for cold-room self-load; without it bots idle honestly (0 walkable cells) instead of wandering a fake board
- `world.welcome` message carries own `conn.id` — must read it BEFORE chat (server's echo self-triggers reactions otherwise)
- Per-bot `this.epoch` generation token guards scheduler chains across disconnect; callbacks bail if epoch changed
- Auto-reconnect: capped exponential backoff+jitter (2s→30s); `started` flag re-arms schedulers once per connection
- NPC disclosure: `bot-` PartyKit conn id triggers existing `worlds.notify.botJoined` "(bot) joined" toast; strips emoji from generated lines
- Live creds (`OPENROUTER_API_KEY`, `TW_ORIGIN`, `OPENROUTER_MODEL`) live in **main-repo `.env`** (gitignored), NOT the worktree root

### Lobby Admin Full-Edit (`engine/world/66-lobby-admin.js`)

- Clicking **Edit Lobby** adds `body.tw-admin-editing` + calls `__tinyworldMode.setBuild()`
- Adds `body.tw-admin-fulledit` (lobby-only; gated by `_laIsLobby()` comparing `WS.getState().world.slug` to `window.__TW_LOBBY_WORLD_SLUG || 'tidewater-bay'`)
- Unlocks: toolbar, palette, flyouts, inspector (`#agent-panel`), radial menu, layers, stamp/asset library, mesh terrain toggle

**Two separate hide systems both gate builder chrome — do not confuse them:**
- `tw-play-mode` (from `__tinyworldMode`, `styles/tiny-world.css:1151`) — removed by `setBuild()`
- `tw-worlds-play` (added by `WS.setPlayChrome(true)` on room enter, `46-worlds-universe.js:67`) — **stays ON while editing**; overrides must beat this

**CSS specificity traps (verified real-browser getComputedStyle, not static reasoning):**
- ID selectors (`body.tw-worlds-play #agent-panel`) have specificity 1,1,1; class-based re-shows LOSE — use `body.tw-admin-fulledit.tw-worlds-play #id` (1,2,1) to win
- Stamps button `#stamp-builder` hidden by pill-trim `:not(#id)` chains (~4 id-weight); override needs matching `:not(#…)` padding
- Toggle panels close via `[hidden]`; `#agent-panel` via `.collapsed` — override with `:not([hidden])` / `:not(.collapsed)` so open/close logic still governs; do not force permanently open
- Mesh terrain toggle surfaced by stylesheet `!important` (beats non-important inline `display:none`; verified real-browser)

**Underside edit** (editable inverted-stepped pyramid) is a separate existing subsystem in `09b-voxel-build-factories.js` + `14-editable-islands-moorings.js`; inspector in `28-generate-panel-agent.js` (`renderEditableIslandPyramidProperties`); needs ZERO new code if admin has `canEditAny()` true. Do NOT conflate with mesh-terrain (its `cellH` clamps ≥0; no underside sculpting).

`tw-worlds-play` intentionally kept ON: its `.controls` trim keeps import/export/reset/clear/world-switch hidden; **Save to Live Lobby** stays the only persistence path.

## Critical Development Rules

- **No emoji** anywhere in UI, code, or output
- **No PNG icons** — SVG glyphs only
- **Never push/commit unless explicitly asked** — "fix it" / "make the changes" = edit files only
- **Never rebuild existing** — lift actual code verbatim; never reimplement/approximate
- **Verify in real browser** — static checks miss runtime/TDZ/visual bugs
- **Never guess** — work from observed facts only
- **Two-attempt rule** — same approach fails twice → stop, verify assumption, ask user
- Admin secret gate: `TINYWORLD_ADMIN_SECRET` must be set; restart `netlify dev` after setting (client works, server silently 403s without it)
- `rm` is aliased interactive in this shell — use `command rm -f`; verify after
- **cwd drifts** into `~/clawd` mirror where edits auto-commit to main — always use absolute paths

## Skybound Roadmap (`plans/ROADMAP-skybound.md`, authored 2026-06-14)

The game vision: a sea-covered planet with floating sky islands; players build/defend/fly; descend to the surface to forage/mine/recruit NPCs; real voxel avatars for all actors.

| Phase | Feature | Status |
|-------|---------|--------|
| 1 | Voxel avatars replace sprite "stripes" (`53-voxel-avatar.js`, `window.makeVoxelAvatar`) | Shipped local, **not pushed to main** |
| 2 | Fly-down to planet surface (camera far + descent + flight collision; `54-fly-down.js`) | In progress |
| 3 | Settlements / NPCs on planet surface (LandscapeEngine placements) | Not started |
| 4 | Real battles (authoritative `battle-<id>` room type; server HP/kills/respawn) | Not started |
| 5 | Crafting loop (`player_items` table + atomic check-and-debit + static recipes) | Not started |
| 6 | Multi-agentic NPCs (AI-driven voxel actors as first-class peers) | Not started |

Key architecture decisions from roadmap:
- Avatars: `makeVoxelAvatar(descriptor) → THREE.Group`; cache geometry by skin-hash; clone-share geometry across identical avatars
- Surface: reuse `initPlanetLandscape` / `LandscapeEngine` (do NOT port voxel-poser's flat sea)
- Battles: model `battle-<id>` room on `isWorldRoom` pattern; not client-trusting `combat.hit` relay
- Crafting: `player_items(profile_id, item_key, qty)` table + atomic spend path first; recipes as static server const
